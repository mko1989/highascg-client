Getting Started Shortcuts

Welcome to HighAsCG Prep Studio

This desktop kit allows production operators and designers to prepare layouts, edit device mappings, configure cabling matrices, and test scenes entirely offline in simulation mode without playout hardware. When ready, plug your stick into the physical studio hardware to auto-sync operations.

How to Flash the HighAsCG Live ISO

Flashing writes the core Linux operating system structure onto the USB stick, allowing the studio rack node to boot from it. We recommend using Balena Etcher for flashing on both Windows and macOS.

01

Load the HighAsCG ISO

Download the official highascg_*.iso build. Open Balena Etcher and click Flash from file to select the ISO image.

Screenshot: Loading ISO in Etcher Click 'Flash from file' and select highascg_amd64.iso
02

Select the Target USB Drive

Click Select target and choose your plugged USB flash drive. CAUTION: Triple check that you select the correct removable USB drive by model and capacity to prevent erasing other data.

Screenshot: Target USB Selection Select the flash drive (minimum 16GB recommended)
03

Flash and Verify

Click Flash!. The OS will prompt for administrator permission. Etcher will burn the partitions and run a verification pass. Once complete, you can close Etcher.

IMPORTANT: Do not format or touch any partitions yet when prompted by Windows/macOS. Proceed to the Partitioning tab first.
Screenshot: Flashing Verification Success Wait for 'Flash Complete!' status banner

Setting up the exFAT Partition (WO-47)

After flashing, you must create a readable exFAT partition in the leftover unallocated space at the end of the stick. This partition carrying your offline simulator files and configurations must be labelled exactly HIGHASCGEXF.

01

Open Disk Management

Press Win + X and select Disk Management. Look for your USB drive under the lower list (labeled Removable). Do not touch NVMe Disk 0.

Screenshot: Disk Management Layout Locate the USB Disk showing black unallocated region at the right
02

Create New Simple Volume

Right-click the black bar showing Unallocated space at the very end of your USB disk, and select New Simple Volume...

Screenshot: Unallocated Right-Click Context Menu Click 'New Simple Volume...' wizard command
03

Format Options & exFAT Label

Follow the wizard. At the formatting step, select File system: exFAT. In Volume label, type exactly HIGHASCGEXF (case-sensitive, no spaces, 11-char limit).

Screenshot: Simple Volume Formatting Settings Ensure exFAT format and 'HIGHASCGEXF' label are typed correctly
04

Seed Folders & App Payload

Open the new drive in File Explorer. Create folders inside: sim/highascg, drop-config, media, templates, and configs. Unzip the HighAsCG release zip directly inside sim/highascg/, so that package.json sits directly under that path.

Screenshot: Root exFAT File Layout Verify E:\sim\highascg\package.json file sits in the tree

What this does (and does not do)

You already flashed the boot ISO. These steps add a second partition for operator files — they do not re-flash the stick.

Replace disk2 in every command with your USB id from step 1. Never use internal disks (disk0, disk1).

Only grey boxes with a Copy button are commands to paste in Terminal. Explanations and sample output are plain text — do not paste those.

1

List external USB disks (read-only)

Safe — only prints information.

diskutil list external physical

Pick the row that matches your stick capacity (e.g. 32 GB). Note the whole-disk id (disk2) — not a slice like disk2s2.

!

No Homebrew parted

brew install parted does not work on macOS (no formula; your brew error is expected). Use Apple’s built-in fdisk below — already on your Mac.

2

Understand the layout (diskutil — read-only)

Safe — only lists partitions on your stick.

diskutil list disk2

macOS often shows only a tiny 0xEF slice (~17 MB) plus “(free space)”. The ~5.5 GB ISO still occupies the start of the stick — that space is not safe for exFAT.

Typical output (example — do not paste):

0: FDisk_partition_scheme *31.5 GB disk2 1: 0xEF 16.8 MB disk2s2 (free space) 31.4 GB -
3

Choose a safe start (MiB / sectors)

In Finder, check ISO size. Round up to the next whole gigabyte, then multiply by 1024 for MiB.

ISO size (approx.)Start at (GiB)EXFAT_START_MIB
≤ 5.5 GB (current build)6 GiB6144
≤ 6.5 GB7 GiB7168
≤ 7.5 GB8 GiB8192

Example: 5.4 GB ISO → 6 GiB → 6144 MiB. When in doubt, pick a larger start.

4

Calculate start/end sectors (read-only)

Safe — prints numbers only. Default start is 6144 MiB (6 GiB). Edit path to your repo if needed.

bash /path/to/highascg/client/tools/live-usb/macos/calc-exfat-mbr-sectors.sh disk2

Write down START_SECTOR and END_SECTOR from the output. You will type those numbers in step 6 (do not paste the whole fdisk session).

Larger ISO: EXFAT_START_MIB=7168 bash …/calc-exfat-mbr-sectors.sh disk2 (copy as one line if needed).

5

Unmount the stick (before editing MBR)

Unmounts volumes on the USB so the partition table can be edited safely.

diskutil unmountDisk disk2
6

Add MBR entry with fdisk (interactive)

Opens an editor on the USB partition table. Use exit to leave without saving if something looks wrong. Use write only after print looks correct.

sudo fdisk -e /dev/rdisk2

Type each line below, press Return after each. Do not paste the whole list at once.

  1. print — review table; note first unused slot (often #2)
  2. edit 2 — use your free slot number if not 2
  3. 07 — partition type (Windows NTFS/exFAT family)
  4. When asked CHS vs sector offsets, choose sectors (not CHS defaults)
  5. Start sector: value of START_SECTOR from step 4 (e.g. 12582912 for 6 GiB)
  6. End sector: value of END_SECTOR from step 4
  7. print — confirm new row; boot ISO slice must be unchanged
  8. write — confirm y if prompted
  9. exit — leave fdisk (not quit, which can write again)

If macOS asks for a reboot after write, answer yes and re-plug the stick before step 7.

7

Verify the new slice (read-only)

Safe — lists disks again. Find the new line (size ~25 GB); note its slice id.

diskutil list disk2

Often disk2s1 or disk2s3 — use your new slice in step 8.

8

Format only the new slice as exFAT

Replace disk2s4 with your slice from step 7 (often disk2s1 or disk2s4). Formats that slice only — not the whole disk2.

After fdisk, macOS may list the slice as Windows_NTFS with no filesystem yet. diskutil eraseVolume often fails with -5343 no such partition — use newfs_exfat instead (built into macOS).

sudo newfs_exfat -v HIGHASCGEXF /dev/rdisk2s4

When done, Finder should show HIGHASCGEXF at /Volumes/HIGHASCGEXF. If it does not mount:

diskutil mount disk2s4
9

Create operator folders

Writes only on the exFAT volume.

mkdir -p "/Volumes/HIGHASCGEXF/sim/highascg" "/Volumes/HIGHASCGEXF/drop-config" "/Volumes/HIGHASCGEXF/media" "/Volumes/HIGHASCGEXF/templates" "/Volumes/HIGHASCGEXF/configs" "/Volumes/HIGHASCGEXF/snapshots/rear-panels"
10

Copy the HighAsCG release

Choose one method. Edit the source path before running.

From unpacked folder:

ditto "/path/to/highascg-main/." "/Volumes/HIGHASCGEXF/sim/highascg"

From tarball:

tar -xzf ~/Downloads/highascg_*.tar.gz -C "/Volumes/HIGHASCGEXF/sim/highascg"

Verify package.json exists:

test -f "/Volumes/HIGHASCGEXF/sim/highascg/package.json" && echo OK
?

Optional: Disk Utility (only if Partition is not greyed out)

Some GPT sticks can use Disk Utility: View → Show All Devices → physical USB → Partition → ExFAT slice named HIGHASCGEXF. If Partition is disabled, use the Terminal steps above.
