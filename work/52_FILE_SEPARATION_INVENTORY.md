# HighAsCG File Separation & Distribution Inventory

This document provides a concrete list of files and directories showing exactly which assets belong to the **Headless Backend (Server Clone)** and which belong to the **Decoupled Frontend (SPA)**. Use this list to configure penguins-eggs exclude patterns or separate git repositories.

---

## 📁 1. Frontend Repository Only (SPA UI client)
These folders and files represent the static visual interface. They are compiled into optimized static HTML, CSS, and JS files by Vite. **They must be completely removed from the headless backend server clone.**

### Directories
- `web/` — The entire directory is frontend-only:
  - `web/components/` — All UI panels and inspectors.
  - `web/lib/` — Frontend client WS/API libraries and math helpers.
  - `web/assets/` — SVG icons, graphic logos, and previs modules.
  - `web/styles/` — Segmented CSS layout design scripts.
- `dist-web/` — The compiled static production build generated by Vite.

### Files
- `vite.config.js` — Vite development server and builder setup.
- `web/index.html` — The main entry point document.
- `web/app.js` — The application main bootstrapper.
- `web/styles.css` — Global CSS styling sheets.

---

## 📁 2. Headless Backend Repository Only (Express Server Engine)
These files constitute the running playout service. **They contain all system intelligence and must stay on the server host, with zero frontend UI dependencies.**

### Directories
- `src/` — All service layers:
  - `src/api/` — REST server route handlers.
  - `src/bootstrap/` — Command-line argument loaders and config builders.
  - `src/caspar/` — TCP AMCP commands broker.
  - `src/config/` — Standard configurations and defaults.
  - `src/device/` — DeckLink, ArtNet, DMX, and GPU controllers.
  - `src/engine/` — Visual layout compilers and take planners.
  - `src/osc/` — OSC network controllers.
  - `src/sampling/` — UDP/ArtNet network receivers.
  - `src/server/` — Express server and native CORS setups.
  - `src/streaming/` — Low-latency WebRTC/NDI preview encoders.
  - `src/utils/` — System memory, inventory, and hardware probes.
- `sim/` — Offline simulation mock logs and files.
- `data/` — Static persistent configuration JSON databases (e.g., `data/known-gpus.json`).
- `scripts/` — ExFAT mounts, systemd loaders, and installation bindings.
- `tools/` — Penguins-eggs tools, live-usb image setup scripts, and drivers.

### Files
- `index.js` — Core bootloader and process manager.
- `highascg.config.json` — The active configuration settings.

---

## 📦 3. Split Strategy for `package.json`
To fully decouple the builds, we split `package.json` dependencies into separate packaging profiles:

### 🖥️ Headless Backend Dependencies (System Host)
```json
{
  "dependencies": {
    "express": "^4.18.2",
    "ws": "^8.13.0",
    "cors": "^2.8.5",
    "busboy": "^1.6.0",
    "artnet": "^1.3.1"
  }
}
```

### 🎨 Frontend SPA Dependencies (Vite Dev Server/Build Host)
```json
{
  "devDependencies": {
    "vite": "^4.4.5"
  },
  "dependencies": {
    "three": "^0.155.0",
    "grapesjs": "^0.21.2",
    "html-to-image": "^1.11.11"
  }
}
```
*(Optional dependencies like three and grapesjs are compiled directly by Vite if installed, or left to browser importmaps if marked external).*
